58/// The main job of the Region is to point to new instructions generated by
59/// vectorization passes. It is the unit that RegionPasses operate on with their
60/// runOnRegion() function.
61///
62/// The region allows us to stack transformations horizontally, meaning that
63/// each transformation operates on a single region and the resulting region is
64/// the input to the next transformation, as opposed to vertically, which is the
65/// common way of applying a transformation across the whole function. This
66/// enables us to check for profitability and decide whether we accept or
67/// rollback at a region granularity, which is much better than doing this at
68/// the function level.
69///
70// Traditional approach: transformations applied vertically for the whole
71// function
72// F
73// +----+
74// | |
75// | |
76// | | -> Transform1 -> ... -> TransformN -> Check Cost
77// | |
78// | |
79// +----+
80//
81// Region-based approach: transformations applied horizontally, for each Region
82// F
83// +----+
84// |Rgn1| -> Transform1 -> ... -> TransformN -> Check Cost
85// | |
86// |Rgn2| -> Transform1 -> ... -> TransformN -> Check Cost
87// | |
88// |Rgn3| -> Transform1 -> ... -> TransformN -> Check Cost
89// +----+
90
